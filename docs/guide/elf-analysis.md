# ELF 结构解析研究

这是本工具的核心底层技术，用于从加固的 native 库（通常为 `libEPIC.so`）中提取关键的加密元数据。

## 研究结果

### .ArmEpic 节区分析
*   **存储位置**：在 ELF 文件的节区头部表（Section Header Table）中。
*   **数据结构**：
    *   **V1 版本**：采用紧凑的二进制结构，包含包名标识及 16 字节的原始 RC4 密钥。
    *   **V2 版本**：采用“密钥 + 加密 JSON”的混合结构。前 16 字节为解密 Key，后续为经过 RC4 加密的 JSON 字符串。

## 解析流程
1.  **架构自适应**：自动识别并处理 ELF32 (ARM) 与 ELF64 (AArch64) 的数据偏移差异。
2.  **字符串表定位**：通过 `e_shstrndx` 找到字符串表，搜索名为 `.ArmEpic` 的节区名。
3.  **节区提取**：根据节区头部（Shdr）记录的 `offset` 和 `size` 提取原始字节流。
4.  **动态解密配置**：提取前 16 字节作为“元密钥”，解密节区剩余部分，获取最终的业务逻辑 JSON 树。
