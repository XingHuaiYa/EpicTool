# Dex 数据恢复研究

针对 Epic 加固中 DEX 文件的保护机制，本项目研究了其 V1 与 V2 版本的差异化加密方案及静态还原路径。

## 研究结果

### Epic V1 保护特征
*   **存储路径**：加密后的 DEX 通常存放于 `assets/Epic_dexs/` 目录下。
*   **加密算法**：采用标准 RC4 流加密。
*   **密钥来源**：密钥动态隐藏在 native 库的 `.ArmEpic` 节区中。

### Epic V2 保护特征
*   **存储路径**：由配置文件中的 `dex_dir` 参数指定，通常位于 `assets/` 的子目录下。
*   **文件后缀**：加密文件通常以 `.epic` 作为后缀。
*   **加密算法**：支持 RC4 (Method 1) 或单字节异或 (Method 0) 两种模式。

## 还原方法
1.  **配置提取**：通过解析 ELF 文件的特定节区获取加固配置信息。
2.  **参数识别**：识别配置中的解密方法标识（Method）与对应的密钥（Key）。
3.  **流式解密**：遍历加密目录，根据文件序号进行流式还原。
4.  **结构验证**：还原后自动校验 DEX 文件头魔数（`64 65 78`），确保数据完整性。
