# Assets 资源还原研究

为了对抗资源提取，Epic V2 引入了资源重命名与加密存储的映射机制。

## 研究结果

### 资源保护机制
*   **重命名映射**：原始资源路径（Logic Path）被映射为随机生成的真实路径（Real Path）。
*   **加密封装**：资源文件内容根据全局配置进行二次加密。
*   **集中配置**：所有的映射关系（Rename Map）均存储在加密的 JSON 配置块中。

## 还原方法
1.  **读取映射表**：从 `asset_rename_map` 中提取逻辑路径与物理路径的对应关系。
2.  **按需解密**：根据 `asset_protection_method` 选择对应的 RC4 或异或解密逻辑。
3.  **目录重构**：解析逻辑路径中的层级关系，递归创建对应的文件夹结构。
4.  **批量还原**：将解密后的原始数据流写回到正确的目录路径下，实现全资源的自动化还原。
