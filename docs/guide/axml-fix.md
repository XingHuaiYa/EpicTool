# Axml 资源修复研究

Epic V2 对 Android 的二进制 XML（Axml）结构进行了破坏性保护，导致常规工具无法直接解析。

## 研究结果

### 破坏手段
*   **魔数篡改**：将 Axml 标准头部魔数修改为 `Epic` (Hex: `45 70 69 63`)。
*   **数据加密**：XML 数据体部分经过 RC4 或异或加密。
*   **结构偏移**：破坏了 Header 的固定结构，使解析器定位失败。

## 修复方法
1.  **特征扫描**：遍历 `/res` 目录，通过文件头部的 `Epic` 标识定位受保护的 XML 文件。
2.  **剥离与解密**：应用从配置中提取的 Axml 专用密钥进行解密。
3.  **结构重组**：将解密后的数据头部强制修正为标准 Axml 魔数（`03 00 08 00`），并重新计算填充资源索引表。
4.  **打包校验**：将修复后的 XML 重新封装，确保其能被反编译工具（如 AXMLPrinter）正常读取。
